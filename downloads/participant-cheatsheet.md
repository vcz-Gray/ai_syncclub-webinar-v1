# AI 싱크클럽 웨비나 참가자 치트시트

> 이 한 장이면 오늘 바로 시작할 수 있습니다.

## 오늘 바로 실행하는 5단계

| 단계 | 할 일 | 소요 시간 |
|------|-------|-----------|
| 1 | 만들고 싶은 서비스를 **한 문장**으로 적는다 | 5분 |
| 2 | PRD 4문항에 답한다 (아래 참고) | 15분 |
| 3 | Stitch에 PRD를 넣고 **첫 화면 1개**를 생성한다 | 10분 |
| 4 | 추가 화면 2개를 더 만들고, Prototypes로 흐름을 연결한다 | 20분 |
| 5 | 결과 스크린샷을 저장하고, 내일 수정 지시를 넣을 메모를 적는다 | 5분 |

**총 소요: 약 55분. 오늘 밤이면 첫 프로토타입이 나옵니다.**

## PRD 4문항 (이것만 답하면 됩니다)

```
1. 누가 쓰나?
   → 핵심 사용자를 딱 1명으로 특정하세요.
   → 예: "소규모 스터디 운영자", "1인 클래스 강사"

2. 뭐가 불편한가?
   → 지금 겪는 구체적인 문제를 한 문장으로.
   → 예: "출석 체크가 매번 카톡으로 흩어진다"

3. 핵심 기능 3개?
   → 반드시 3개만. 4개 이상이면 하나를 잘라야 합니다.
   → P0(반드시 필요) 3개만 남기고 나머지는 P1으로 미루세요.

4. 화면은 몇 개인가?
   → 3~4개가 적당합니다. 각 화면의 목적을 한 줄로 적으세요.
   → 예: 홈(대시보드), 입력(등록 폼), 결과(목록), 상세(세부 정보)
```

---

## 오늘의 목표 (착각하지 마세요)

### 오늘의 목표가 아닌 것:
- ~~완성품 만들기~~
- ~~예쁜 디자인 만들기~~
- ~~수익 모델 정하기~~
- ~~10가지 기능 넣기~~

### 오늘의 진짜 목표:
- 내 아이디어를 **한 문장**으로 설명하기
- 핵심 사용자와 문제를 **좁히기**
- 첫 화면과 흐름을 **눈으로 보기**

**성공 기준: "화면이 나왔는가?" — 이것 하나입니다.**

## Stitch 프롬프트 빠른 시작

### 첫 프롬프트 (복사해서 [ ] 부분만 바꾸세요)

```
한국어 기반 웹 서비스 UI를 만들어줘.
대상 사용자는 [핵심 사용자]이고,
핵심 문제는 [문제]야.
필수 기능은 [기능1], [기능2], [기능3]이고,
화면은 [홈], [입력], [결과] 3개로 구성해줘.
실제 서비스 초안처럼 만들어줘. 깔끔하고 신뢰감 있는 디자인.
```

### 결과가 안 좋을 때 → 이렇게 수정 지시하세요

```
정보가 너무 많아. 사용자가 처음 봤을 때 핵심 행동이 바로 보이도록 단순화해줘.
카드 수를 줄이고, 가장 중요한 버튼을 눈에 띄게, 제목과 설명의 위계를 분명하게.
```

## 막혔을 때 즉시 해결법

| 증상 | 해결법 |
|------|--------|
| 아이디어가 너무 크다 | 핵심 사용자 1명, 핵심 문제 1개로 축소 |
| 기능이 너무 많다 | P0 3개만 남기고 나머지 전부 P1으로 |
| 화면이 안 나온다 | 페이지 수를 먼저 정하고, 각 페이지 목적을 한 줄로 |
| 결과가 엉뚱하다 | 레퍼런스 앱을 직접 언급. "에어비앤비 검색 화면처럼" |
| 디자인이 촌스럽다 | "다크 테마, Material 3, 둥근 카드" 같은 스타일 키워드 추가 |
| 수정해도 안 고쳐진다 | 새로 생성하지 말고 **채팅으로 부분 수정** 지시 |
| 다음 단계가 막막하다 | post-webinar-next-steps.md의 Day 1-3-7 플랜을 따라가기 |

## 가장 쉬운 화면 순서

```
1. 홈/랜딩 → 서비스 목적이 한눈에 보이는 화면
2. 입력/등록 → 사용자가 정보를 넣는 화면
3. 결과/목록 → 처리 결과나 데이터를 보는 화면
4. 상세 (선택) → 개별 항목의 세부 정보
```

---

## 기억할 문장

- **PRD는 길게 쓰는 문서가 아니라 AI에게 보여줄 설계도다.**
- **Stitch 프롬프트는 멋진 문장이 아니라 우선순위가 분명한 문장이다.**
- **프롬프트를 개선하면 결과가 극적으로 달라진다. 최소 3~5번은 바꿔가며 시도하라.**
- **한 번에 하나의 기능만 만들어라. "전체 앱"이 아니라 "로그인 화면"부터.**
- **AI 출력물은 반드시 검증하라. "바이브"는 생성 과정이지, 검증을 건너뛴다는 뜻이 아니다.**
