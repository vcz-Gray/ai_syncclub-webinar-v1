# AI 싱크클럽 웨비나 참가자 치트시트

## 오늘 바로 해야 할 것

1. 만들고 싶은 서비스 하나만 고른다.
2. 아래 4문항으로 PRD를 쓴다.
3. Stitch에 넣을 첫 프롬프트를 만든다.
4. 화면 1개가 아니라 화면 3개까지 본다.
5. 내일 다시 열어 1차 수정 지시를 넣는다.

## PRD 4문항

1. 누가 쓰나?
2. 뭐가 불편한가?
3. 핵심 기능 3개는?
4. 화면은 몇 개인가?

## 오늘의 목표

- 완성품 만들기
- 예쁜 디자인 만들기
- 수익 모델 정하기

위 3개가 아니라 아래 3개입니다.

- 내 아이디어를 한 문장으로 설명하기
- 핵심 사용자와 문제를 좁히기
- 첫 화면과 흐름을 눈으로 보기

## 가장 쉬운 시작 순서

1. 랜딩/홈
2. 핵심 입력 화면
3. 결과/목록 화면

## 막혔을 때

- 기능이 너무 많다 → 3개만 남긴다
- 화면이 너무 많다 → 3개만 본다
- 설명이 길다 → 한 문장씩 끊는다
- 결과가 엉뚱하다 → 사용자/문제/핵심 기능을 다시 써준다

## 기억할 문장

- PRD는 길게 쓰는 문서가 아니라 AI에게 보여줄 설계도다.
- Stitch 프롬프트는 멋진 문장이 아니라 우선순위가 분명한 문장이다.
- 오늘의 성공 기준은 “화면이 나왔는가”다.
