## 절대 하지 말아야 할 것

| 금지 행동 | 이유 |
|-----------|------|
| 첫 주에 기능을 10개로 늘리기 | P0 3개도 안 다듬었는데 늘리면 전부 어중간해집니다 |
| 디자인 완성도에만 집착하기 | 구조가 맞아야 디자인도 의미가 있습니다 |
| 주변 의견을 너무 많이 받기 | 1명의 피드백이면 충분합니다. 3명 이상이면 방향이 흔들립니다 |
| 바로 개발 견적부터 알아보기 | 프로토타입이 검증되기 전에 개발비를 쓰면 돈을 버리는 겁니다 |
| 첫 결과에 실망해서 포기하기 | 프롬프트를 3~5번 바꾸면 결과가 극적으로 달라집니다 |
| 처음부터 완벽한 코드 만들려 하기 | 프로토타입은 "버릴 수 있는 코드"입니다 |
