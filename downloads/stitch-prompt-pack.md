# Stitch 프롬프트 팩

## 1. 기본 생성 프롬프트

```text
한국어 기반 웹 서비스 UI를 만들어줘.
대상 사용자는 [핵심 사용자]이고,
핵심 문제는 [문제]야.
필수 기능은 [기능1], [기능2], [기능3]이고,
화면은 [홈], [입력], [결과] 3개로 구성해줘.
너무 복잡하지 않게, 실제 서비스 초안처럼 만들어줘.
```

## 2. 랜딩 페이지 중심 프롬프트

```text
이 서비스의 첫 인상을 보여주는 랜딩 페이지를 만들어줘.
상단에는 서비스 이름과 한 줄 가치 제안,
중간에는 핵심 기능 3개,
하단에는 시작 버튼과 간단한 FAQ가 보이게 해줘.
너무 마케팅 페이지처럼 과장하지 말고, 실제 서비스 진입점처럼 구성해줘.
```

## 3. 내부 업무 도구 프롬프트

```text
작은 팀이 쓰는 내부 업무 도구 화면을 만들어줘.
왼쪽에는 상태 필터,
가운데에는 항목 목록,
오른쪽에는 상세 패널이 있는 3단 레이아웃으로 구성해줘.
관리자 대시보드처럼 보이되 과하게 복잡하지 않게 해줘.
```

## 4. 멀티스크린 생성 프롬프트

```text
이 서비스의 사용자 흐름을 볼 수 있게 4개 화면을 만들어줘.
1. 홈
2. 입력/등록
3. 결과/목록
4. 상세 보기
각 화면은 같은 디자인 시스템을 공유해야 하고,
사용자가 어떻게 이동하는지 자연스럽게 이어지게 해줘.
```

## 5. 수정 지시 프롬프트

```text
전체적으로 정보가 많아 보여.
사용자가 처음 봤을 때 핵심 행동이 더 잘 보이도록 단순화해줘.
카드 수를 줄이고, 가장 중요한 버튼을 더 눈에 띄게 만들고,
제목과 설명의 위계를 더 분명하게 해줘.
```

## 6. 톤 조정 프롬프트

```text
좀 더 신뢰감 있는 톤으로 바꿔줘.
스타트업 홍보 느낌보다 실제 서비스 프로토타입처럼 보이게 하고,
장식은 줄이고 구조와 정보 위계를 더 또렷하게 만들어줘.
```

## 7. Prototypes 연결 프롬프트

```text
이제 만든 4개 화면을 Prototypes 기준으로 연결해줘.
홈에서 등록 화면으로,
등록 화면에서 결과 목록으로,
목록에서 상세 보기로 이동하는 기본 흐름을 보이게 해줘.
버튼과 클릭 포인트가 자연스럽게 이어지도록 구성해줘.
```

## 8. 바로 복사해서 쓰는 서비스별 예시

### 스터디 운영 도구

```text
스터디 운영자가 참여자 출석과 공지를 한 번에 관리하는 웹 서비스 UI를 만들어줘.
핵심 기능은 참여자 초대, 출석 체크, 공지 기록이야.
홈, 출석 체크, 공지 기록, 참여자 관리 화면을 만들어줘.
```

### 예약 문의 서비스

```text
소규모 클래스 운영자를 위한 예약 문의 서비스 UI를 만들어줘.
핵심 기능은 클래스 목록 보기, 예약 신청, 신청 확인이야.
랜딩, 클래스 목록, 신청 폼, 신청 완료 화면으로 구성해줘.
```

### 콘텐츠 검수 대시보드

```text
작은 콘텐츠 팀이 검수 요청을 관리하는 내부 대시보드를 만들어줘.
핵심 기능은 요청 등록, 상태 변경, 담당자 필터야.
목록, 상세, 통계 요약 화면을 만들어줘.
```
