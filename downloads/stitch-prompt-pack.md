## 1. 기본 생성 프롬프트 (만능)

```text
한국어 기반 웹 서비스 UI를 만들어줘.
대상 사용자는 [핵심 사용자]이고,
핵심 문제는 [문제]야.
필수 기능은 [기능1], [기능2], [기능3]이고,
화면은 [홈], [입력], [결과] 3개로 구성해줘.
너무 복잡하지 않게, 실제 서비스 초안처럼 만들어줘.
깔끔하고 신뢰감 있는 디자인, 둥근 카드, 밝은 배경.
```

---

## 2. 랜딩 페이지 프롬프트

```text
이 서비스의 첫 인상을 보여주는 랜딩 페이지를 만들어줘.
상단: 서비스 이름 + 한 줄 가치 제안 + 시작 버튼
중간: 핵심 기능 3개 (아이콘 + 제목 + 설명)
하단: 사용자 후기 또는 간단한 FAQ + CTA 반복
너무 마케팅 느낌 말고, 실제 서비스 진입점처럼 구성해줘.
모던하고 전환율 높은 디자인. [참고 앱] 스타일 참고.
```

---

## 3. 내부 업무 도구 프롬프트

```text
작은 팀이 쓰는 내부 업무 도구 화면을 만들어줘.
왼쪽: 상태 필터 사이드바
가운데: 항목 목록 (테이블 또는 칸반)
오른쪽: 상세 패널 (사이드 시트)
관리자 대시보드처럼 단정하고 실무적으로. Linear/Notion 참고.
상단에 KPI 카드 3~4개, 다크 테마, 모노스페이스 숫자.
```

---

## 4. 멀티스크린 생성 프롬프트

```text
이 서비스의 사용자 흐름을 볼 수 있게 4개 화면을 만들어줘.
1. 홈 — 서비스 목적이 한눈에 보이는 진입점
2. 입력/등록 — 사용자가 정보를 넣는 화면
3. 결과/목록 — 처리 결과나 데이터를 보는 화면
4. 상세 보기 — 개별 항목의 세부 정보
각 화면은 같은 디자인 시스템을 공유해야 하고,
사용자가 어떻게 이동하는지 자연스럽게 이어지게 해줘.
내비게이션은 일관성 있게 유지.
```

---

## 5. 수정 지시 프롬프트 (만능)

```text
전체적으로 정보가 많아 보여.
사용자가 처음 봤을 때 핵심 행동이 바로 보이도록 단순화해줘.
- 카드 수를 [N개]에서 [M개]로 줄여줘
- 가장 중요한 버튼을 더 눈에 띄게 만들어줘
- 제목과 설명의 위계를 더 분명하게 해줘
- [참조 앱] 스타일을 참고해서 톤을 맞춰줘
```

---

## 6. 톤/스타일 조정 프롬프트

```text
좀 더 [톤]으로 바꿔줘.
장식은 줄이고 구조와 정보 위계를 더 또렷하게 만들어줘.

# 톤 옵션 (골라 쓰세요):
- "신뢰감 있는" — 관리자 도구, 내부 시스템
- "따뜻하고 접근하기 쉬운" — 커뮤니티, 교육
- "깔끔하고 전문적인" — SaaS, 예약 시스템
- "다크 테마, 데이터 밀도 높은" — 대시보드, 트래커
- "밝고 경쾌한" — 개인 앱, 루틴 도구
```

## 7. Prototypes 연결 프롬프트

```text
이제 만든 4개 화면을 Prototypes 기준으로 연결해줘.
홈에서 등록 화면으로,
등록 화면에서 결과 목록으로,
목록에서 상세 보기로 이동하는 기본 흐름을 보이게 해줘.
버튼과 클릭 포인트가 자연스럽게 이어지도록 구성해줘.
뒤로가기와 내비게이션 흐름도 포함해줘.
```

## 8. 디자인 시스템 지정 프롬프트

```text
아래 디자인 시스템을 적용해줘:
- 색상: [메인색] + [강조색] + [배경색]
- 폰트: [예: Pretendard, 본고딕]
- 카드: 둥근 모서리(16px), 그림자 있음
- 버튼: pill 모양, 메인색 배경, 흰색 글자
- 아이콘: [예: Phosphor, Lucide] 스타일
- 전체 톤: [예: Material 3, Apple HIG] 참고
```

## 9. 앱 유형별 즉시 사용 프롬프트

### 대시보드

```text
[서비스명] 관리자 대시보드.
좌측 사이드바 내비게이션, 상단 KPI 카드 4개,
메인 영역에 테이블 뷰. Linear 스타일, 다크 테마.
상태별 색상 체계 적용 (빨강/주황/노랑/초록).
```

### 마켓플레이스

```text
[카테고리] 마켓플레이스.
상품 그리드(썸네일+가격+평점), 카테고리 필터 사이드바,
상품 상세, 장바구니. 에어비앤비/쿠팡 참고.
모바일 반응형, 카드 기반.
```

### 예약 시스템

```text
[서비스] 예약 시스템.
서비스 선택 → 날짜/시간 선택(캘린더) → 정보 입력 → 확인.
Calendly 참고, 깔끔하고 전문적인 톤.
가능 시간대를 초록색으로 표시.
```

### 커뮤니티

```text
[주제] 커뮤니티.
피드(카드형 게시물), 글 작성, 프로필, 알림.
따뜻한 색감, 둥근 UI, 접근하기 쉬운 느낌.
좋아요/댓글 수 표시, 카테고리 탭.
```

## 10. 서비스별 즉시 사용 예시

### 스터디 운영 도구

```text
스터디 운영자가 참여자 출석과 공지를 한 번에 관리하는 웹 서비스.
핵심 기능: 참여자 초대, 출석 체크(토글), 회차별 공지 기록.
화면: 홈(출석률 차트), 출석 체크, 공지 기록, 참여자 관리.
밝은 배경, 깔끔한 카드 UI. 교육 플랫폼 느낌.
```

### 예약 문의 서비스

```text
소규모 클래스 운영자를 위한 예약 문의 서비스.
핵심 기능: 클래스 목록(잔여석 표시), 예약 신청, 신청 확인.
화면: 랜딩, 클래스 목록, 신청 폼, 신청 완료.
전문적이고 신뢰감 있는 디자인. 인기 클래스에 태그.
```

### 콘텐츠 검수 대시보드

```text
작은 콘텐츠 팀이 검수 요청을 관리하는 내부 대시보드.
핵심 기능: 요청 등록, 상태 변경(대기→진행→완료), 담당자 필터.
화면: 목록(칸반 뷰), 상세(사이드 패널), 통계 요약.
관리자 도구 스타일, 단정하게. 마감 임박 빨간색 강조.
```

## 프롬프트 작성 핵심 원칙

1. **맥락을 먼저**: 대상 사용자 + 화면 목적 + 핵심 데이터
2. **스타일을 구체적으로**: 색상, 테마, 카드 형태, 디자인 시스템
3. **레퍼런스 앱을 언급**: "에어비앤비처럼", "Linear 스타일로"
4. **한 번에 한 가지 변경**: 여러 수정을 한꺼번에 넣지 말 것
5. **5000자 이하**: 너무 길면 AI가 핵심을 놓침
6. **채팅으로 반복 개선**: 처음부터 다시 생성하는 것보다 부분 수정이 효율적
