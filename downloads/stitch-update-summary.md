# Stitch 최신 업데이트 요약

> 2026년 3월 기준, Google Stitch에 적용된 핵심 업데이트와 참가자에게 미치는 영향을 정리했습니다.

---

## 핵심 업데이트 2가지

### 1. Gemini 3 적용

| 항목 | 변화 |
|------|------|
| **생성 품질** | 더 정교한 레이아웃, 일관된 디자인 시스템, 세밀한 타이포그래피 |
| **한국어 이해** | 한국어 프롬프트의 맥락 파악이 크게 개선됨. "관리자 도구처럼 단정하게" 같은 뉘앙스도 반영 |
| **일관성** | 같은 프롬프트로 재생성해도 품질 편차가 줄어듦 |
| **속도** | 생성 속도 향상. Standard 모드에서 체감 차이 큼 |

### 2. Prototypes 추가

| 항목 | 변화 |
|------|------|
| **흐름 검토** | 단일 시안이 아니라 여러 화면을 이어서 볼 수 있음 |
| **클릭 연결** | 홈 → 입력 → 결과 같은 사용자 여정을 버튼 클릭으로 탐색 |
| **PRD 검증** | PRD에서 정의한 화면 수와 이동 흐름을 바로 검토 가능 |
| **공유** | 프로토타입을 링크로 공유해서 피드백을 받을 수 있음 |

---

## 모드별 차이

| 모드 | 모델 | 월 사용량 | 최적 용도 |
|------|------|-----------|-----------|
| **Standard** | Gemini 2.5 Flash | 350회 | 빠른 아이디어 검증, 반복 수정, 다양한 시도 |
| **Experimental** | Gemini 2.5 Pro | 50회 | 이미지 기반 정밀 디자인, 최종 시안 |

**전략**: Standard로 마음껏 시도 → 방향이 잡히면 Experimental로 마무리.

---

## 참가자에게 중요한 이유

1. **"예쁜 화면 한 장"이 아니라 "사용자 흐름"을 볼 수 있습니다.**
   - 이전: 화면 1개를 보고 "이런 느낌이구나" 정도
   - 이후: 홈→입력→결과→상세를 실제로 클릭하며 흐름 검증

2. **PRD의 가치가 배가됩니다.**
   - PRD에서 정의한 화면 수와 흐름을 바로 프로토타입으로 확인
   - "PRD 없이 만든 것"과 "PRD 넣고 만든 것"의 차이가 더 극명해짐

3. **피드백 수집이 쉬워집니다.**
   - 정적 이미지 대신 클릭 가능한 프로토타입을 공유
   - "5초 안에 뭐 하는 서비스인지 이해되나?" 테스트가 현실적으로 가능

4. **다음 단계(AI Studio Build)로의 전환이 자연스럽습니다.**
   - Stitch 프로토타입으로 방향 검증 → AI Studio에서 실제 코드 생성

## Prototypes 활용 프롬프트

```text
이제 만든 4개 화면을 Prototypes 기준으로 연결해줘.
홈에서 등록 화면으로, 등록에서 결과 목록으로,
목록에서 상세 보기로 이동하는 기본 흐름을 보이게 해줘.
버튼과 클릭 포인트가 자연스럽게 이어지도록 구성해줘.
뒤로가기 흐름도 포함해줘.
```

## 발표 멘트용 한 줄 정리

- "이제는 화면을 만드는 단계에서 끝나지 않고, **연결된 프로토타입까지 빠르게 볼 수 있습니다.**"
- "Gemini 3 덕분에 한국어 프롬프트의 결과 품질이 **체감될 정도로** 올라갔습니다."
- "PRD를 넣으면 Prototypes까지 **자동으로 흐름이 연결**됩니다."
