CLAUDE.md 작성 가이드

프로젝트의 "헌법". AI가 매 세션마다 가장 먼저 읽는 파일을 제대로 쓰는 법입니다.

반드시 포함할 항목

  • 프로젝트 컨텍스트 — 기술 스택, 목적을 1~2줄로
  • 필수 명령어 — dev, test, build, lint, deploy
  • 디렉토리 맵 — 코드가 어디에 있는지
  • 비기본 컨벤션 — "Zustand 사용, Redux 금지" 같은 규칙
  • 흔한 실수 방지 — "마이그레이션 먼저 실행할 것"
  • 참조 문서 — "@docs/api.md" + "Read when: API 수정 시"

파일 계층 구조

  • ./CLAUDE.md — 프로젝트 루트 (주 파일, Git 커밋)
  • .claude/rules/*.md — 자동 로드 규칙 (Git 커밋)
  • subfolder/CLAUDE.md — 디렉토리별 특화 (Git 커밋)
  • ./CLAUDE.local.md — 개인 설정 (Git 미포함)
  • 200~300줄 이내로 유지, WHY/WHAT/HOW 구조
  • IMPORTANT, YOU MUST 키워드로 강조

CLAUDE.md 템플릿 예시

# MyProject SaaS 플랫폼 — Next.js 15, React 19, TypeScript 5.6, Supabase ## Commands pnpm dev # 개발 서버 pnpm test # 테스트 (Vitest) pnpm build # 프로덕션 빌드 ## Project Structure src/components/ # React 컴포넌트 (PascalCase) src/routes/ # API 엔드포인트 src/models/ # DB 스키마 ## Rules - IMPORTANT: 함수형 컴포넌트 + hooks만 사용 - YOU MUST: TypeScript strict mode - MUST NOT: Redux, 인라인 스타일, 타입 미지정 props

커스텀 슬래시 커맨드 만들기

자주 쓰는 긴 프롬프트를 /커맨드로 등록하는 방법입니다.

기본 규칙

  • .claude/commands/ 디렉토리에 마크다운 파일 생성
  • 파일명이 곧 커맨드명: commit.md → /commit
  • 하위 디렉토리로 네임스페이스: posts/new.md → /posts:new
  • $ARGUMENTS 플레이스홀더로 유연한 입력 지원
  • 동사-명사 형식으로 일관된 네이밍

실전 커맨드 예시

  • /commit — 표준화된 Git 커밋 메시지 생성
  • /test — 보일러플레이트 테스트 파일 자동 생성
  • /review — 보안/성능/접근성 전체 코드 감사
  • /build — 프로젝트 빌드 후 에러 자동 수정
  • /posts:publish — draft 해제, 검증, Git push

TRD (기술 요구사항 문서) 작성 가이드

PRD가 "방향"이라면, TRD는 "만들기 위한 설명서"입니다. AI 코딩 도구에 넣을 때 최적화된 구조입니다.

TRD 필수 항목

  • 목적 — 프로젝트 존재 이유를 한 줄로
  • 기술 스택 — Frontend, Backend, 배포 환경
  • 핵심 기능별 상세 — 입력/출력/엣지 케이스/수용 기준
  • 코딩 컨벤션 — 함수형, 파일명 규칙, 필수 패턴
  • 참고 화면 — 스크린샷 또는 Figma 링크
  • Few-Shot 예시 — 입출력 시나리오 2~3개 (AI 품질 대폭 향상)

핵심 원칙

  • 자연어(마크다운)로 시작하고, 필요 시 JSON/OpenAPI로 진화
  • AI는 추론하지 않으므로 엣지 케이스를 직접 명시
  • "정확"의 정의를 측정 가능한 조건으로 작성
  • 아키텍처 의도를 기능 요구사항과 함께 문서화
  • 코딩 패턴과 스타일을 프로젝트별로 명시
  • 버전 관리 연동으로 AI에게 변경 히스토리 제공

TRD 핵심 기능 작성 예시

### 1. 사용자 인증 - 입력: 이메일 + 비밀번호 - 출력: JWT 토큰 + 대시보드 리다이렉트 - 엣지 케이스: 중복 이메일, 비밀번호 8자 미만, 네트워크 오류 - 수용 기준: 로그인 후 3초 내 대시보드 표시 ### 2. 대시보드 - 입력: 인증된 사용자 세션 - 출력: KPI 카드 4개 + 최근 활동 테이블 - 엣지 케이스: 데이터 없음 상태, 로딩 상태, API 오류 - 수용 기준: 첫 렌더링 2초 이내, 빈 상태 안내 문구 표시

AI 코딩 프롬프트 작성 가이드

비개발자도 AI 코딩 도구에서 좋은 결과를 얻는 프롬프트 작성법입니다.

Plan-Before-Code

가장 중요한 원칙. "코드 작성하지 마. 계획만 세워줘." → 검토 → 승인 → "좋아, 이제 코딩 시작해." 이 3단계가 결과 품질을 극적으로 올립니다.

구체적 예시와 함께

"폼 만들어줘"가 아니라 "이름, 이메일, 메시지 필드가 있는 문의 폼. 제출 후 확인 메시지 표시. 흰색 배경, 파란색 제출 버튼." 스크린샷 첨부가 가장 효과적.

Goal-Constraints-Format

Goal: 상품 목록 페이지 만들기
Constraints: 모바일 대응, 외부 API 없음, 최대 12개
Format: HTML, CSS, JS 별도 파일

점진적 빌드와 테스트

한 번에 전체가 아니라 기능 하나씩 만들고 테스트. 완료 전에 "폼이 실제로 제출되는지 확인해줘" 같은 자가 검증을 요청하세요.

파일 직접 참조

"config.json과 styles.css를 읽고 우리 설정을 이해해." 말로 설명하기 어려우면 디자인 캡처를 드래그앤드롭으로 보여주세요.

Extended Thinking 활용

Claude Code에서 Ctrl+O로 토글. 아키텍처 결정, 버그 분석, 트레이드오프 판단에 활용. "think hard"라고 말하면 더 깊이 추론합니다.

AI 친화적 프로젝트 구조

AI 코딩 도구가 코드를 잘 이해하고 생성하도록 프로젝트를 구성하는 법입니다.

핵심 원칙

  • 파일당 500줄 미만 유지 (컨텍스트 윈도우 관리)
  • 플랫한 계층 구조 (깊은 중첩 지양)
  • 설명적 파일명: goal-tracker.js, user-profile.tsx
  • 파일 상단 요약: // 사용자 목표 처리; Duolingo API 연동
  • 모듈화: 파일당 하나의 관심사
  • 작업 간 /clear: 새 작업 시 컨텍스트 초기화

권장 폴더 구조

  • project-root/
  •   CLAUDE.md      # AI 지시사항
  •   README.md     # 프로젝트 개요
  •   docs/          # 기획서, 와이어프레임
  •   src/components/ # 재사용 UI 모듈
  •   src/api/       # 엔드포인트
  •   src/app.js     # 진입점
  •   tests/         # 테스트
  •   deploy/        # CI/CD 스크립트

AI와 UI/UX 반복 개선 워크플로

디자인에서 코드까지, AI 도구를 활용한 5단계 반복 개선 프로세스입니다.

1. 요구사항 정의

Claude로 PRD/사용자 플로우 초안을 만듭니다. "코드 작성하지 말고 계획만 세워줘."

2. 초기 프로토타입

Stitch로 UI 초안, AI Studio Build로 작동하는 코드를 빠르게 생성합니다.

3. 반복 정제

길고 상세한 프롬프트로 정확도를 올립니다. 한 번에 한 화면씩 반복합니다.

4. 테스트 & 검증

사용성 테스트, 접근성(WCAG) 확인. 주변 1명에게 보여주고 피드백 수집.

5. 코드 핸드오프

생성된 코드를 내보내거나 GitHub에 연동. 인간 폴리시로 마무리합니다.

핵심 원칙

로우 파이 스케치부터 시작. 스크린샷이 가장 효과적. AI 프로토타입은 항상 사람이 마무리.
